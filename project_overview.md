# Project Overview: Dungeon & 4X Strategy Simulation Game

このドキュメントは、AI（Fable 5 / Sonnet 5）がプロジェクトの全体像、設計思想、および現在の開発進捗を迅速に把握し、自律的な開発・デバッグを行うためのコンテキストファイルです。

---

## 1. ゲーム概要 (Game Concept)
『Civilization』シリーズに代表される4X（Explore:探索、Expand:拡張、Exploit:開発、Exterminate:殲滅）戦略要素を融合させた、独自のダンジョン経営シミュレーションゲーム。

- **コアゲームループ**: 
  1. ダンジョン内にトラップや宝箱、部屋をグリッドベースで建設する。
  2. 来襲する冒険者AIの行動から「感情（リソース）」を回収・プールする。
  3. 清算したリソース（DPなど）を使い、技術開発ツリーを進め、魔王のステータス成長や種族進化を行う。
  4. 金沢市を舞台とした地上世界マップへ領域を拡張し、他魔王や強敵との4X戦略レイヤーでの覇権争いに挑む。

## 2. 現在の開発ステータス (Roadmap & Progress)
プロジェクトは現在、**【フェーズ3：魔王の進化と技術ツリー】** から **【フェーズ4：地上マップと戦略レイヤー】** の移行・交差点にあります。

### 実装済みの主要機能 (Core Systems Implemented)
- **グリッドベース配置システム**: ダンジョン内の2Dタイル配置および内政基盤。
- **冒険者AI（最短経路探索）**: BFS（幅優先探索）アルゴリズムを用いた、冒険者の自律的なオブジェクト踏破・巡回ロジック。
- **感情プール＆清算システム**: 冒険者の感情（恐怖、歓喜など）を資源としてプール・清算するコア経済システム。
- **技術開発ツリー（土台）**: 魔王の成長やダンジョン拡張の前提となるツリー構造。
- **知名度連動型ウェーブシステム**: ダンジョンの知名度（悪名）に応じて確率的に強敵（ゾンビ防衛などを含む）が来襲する防衛ゲーム的要素。

### 直近のターゲットタスク (Next Goals)
- **魔王の分岐進化システム**: パラメータ成長に応じた種族の分岐進化ロジックの実装。
- **戦略レイヤーの拡張**: 金沢市を舞台にした地上マップの導入と、DP消費による領域（階層）拡張システムの基盤構築。
- **ゲームループの深化**: ターン制（完全リアルタイムからの一部移行）の導入、冒険者の職業（ジョブ）と探索目的の多様化。

## 3. 重要クラス・アーキテクチャの設計思想
ソースコードを読み込み・編集する際は、以下の既存システムとの密結合を壊さないよう注意してください。

- **`DungeonManager` / `GridSystem` (仮称)**:
  ダンジョンのタイル情報や建設オブジェクトを管理。BFSによる経路探索のベースとなるノードマップを保持しています。
- **`AdventurerAI` (仮称)**:
  BFSを用いてゴール（宝箱や出口）までの最短経路をリアルタイム計算。状態遷移（FSM）や職業別の挙動拡張が予定されています。
- **`EmotionManager` / `ResourceManager` (仮称)**:
  冒険者から発生した感情を清算し、開発用リソースへ変換するトリガーを管理。
- **`TechTreeManager` / `DemonKing` (仮称)**:
  魔王のステータスおよび解放された技術、進化フラグの管理。

## 4. AI（Fable）への開発指示
1. **既存ロジックの尊重**: 
   特にBFS経路探索や感情清算システムはゲームの根幹です。これらを変更・拡張する場合は、必ず事前に依存関係をスキャンし、破壊的変更を避けてください。
2. **自律デバッグの実行**:
   コード編集後は、Unity MCPのコンソール監視機能を用い、コンパイルエラーや実行時例外が発生した場合は自律的に修正ループを回してください。
3. **作業ログの更新**:
   作業開始時にルートに `dev_log.md` を作成し、ステップごとの進捗や設計判断を記録しながら並走してください。
